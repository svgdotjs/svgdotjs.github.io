# Parents

The `SVG.Parent` prototype is the base wrapper for all elements that can contain other elements. `SVG.Parent` inherits directly from the lowest level of all SVG.js prototypes: `SVG.Element`.

`SVG.Container` adds another level to the parent inheritance stack. Where `SVG.Parent` brings some low level methods like `add()`, `remove()` and `has()` to name a few, `SVG.Container` can and should be used if you want to add your won methods. That way the `SVG.Parent` prototype remains clean. Therefore you should always inherit from `SVG.Container` when implementing your own parent elements.

The parent inheritance stack is: `SVG.Element` > `SVG.Parent` > `SVG.Container`.
